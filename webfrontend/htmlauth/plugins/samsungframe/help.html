Samsung Frame TV Plugin – Help

Overview

This plugin provides two-way local integration between a Samsung The Frame TV and Loxone via MQTT. No cloud or SmartThings account is required — all communication is direct over your local network.

Initial Setup

  1. Enter the TV IP address in the Configuration section and click Save Configuration. The TV's MAC address will be discovered automatically.
  2. Make sure the TV is powered on and displaying a picture.
  3. Click Start Pairing. A popup will appear on the TV — accept it within 30 seconds. The pairing token is saved and reused automatically.
  4. The monitor daemon starts automatically when LoxBerry boots. After pairing, it will begin publishing TV state to MQTT.

TV States

StateMeaning
offTV is in standby
artTV is on and Art Mode is active
onTV is on and showing normal content

State is published (retained) to the configured MQTT state topic.

MQTT Commands

Publish any of the following payloads to the command topic:

PayloadAction
power_onWake TV from standby (WebSocket or Wake-on-LAN)
power_offHold power button 3 s — powers off the TV
art_onEnable Art Mode
art_offDisable Art Mode
key_KEY_MUTESend the MUTE key
key_KEY_VOLUPSend the VOLUME UP key
key_KEY_VOLDOWNSend the VOLUME DOWN key
key_XXXXSend any raw Samsung remote key

Loxone Integration

In Loxone Config, use a Virtual Input (text input) listening on the state topic to react to TV state changes. Use a Virtual Output publishing to the command topic to send commands from Loxone.

Troubleshooting

  • Pairing fails / no popup on TV: Ensure developer mode is enabled on the TV (Settings → Support → Device Care → Developer Mode).
  • Art mode not detected: Art mode API requires a 2024 or later firmware. Update the TV firmware and retry.
  • State stuck at "unknown": The daemon may not be running. Check the log at /opt/loxberry/log/plugins/samsungframe/monitor.log.
  • Wake-on-LAN not working: Ensure the MAC address is correct and that the TV is on wired Ethernet.
